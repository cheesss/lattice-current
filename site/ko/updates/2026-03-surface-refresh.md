# 2026-03: 제품 표면 재구성, Replay Studio, 공유 인텔리전스 패브릭

## 무엇이 바뀌었나

- 모든 패널이 한 화면에서 경쟁하던 구조를 줄이고 워크스페이스 모드 중심 셸로 재구성했습니다
- 아래 전용 허브를 전면에 올렸습니다
  - `Briefing Desk`
  - `Research Desk`
  - `Replay Studio`
  - `Graph Studio`
- 로컬 웹 런타임에서도 리플레이와 스케줄러를 실행할 수 있게 했습니다
- `Data Flow Ops`와 자동화 거버넌스를 추가해 파이프라인 상태, blocker, 지연, 보존 상태를 더 쉽게 볼 수 있게 했습니다
- 뉴스, 카테고리 피드, 클러스터, 분석 아티팩트를 공유 인텔리전스 패브릭으로 저장하고 여러 허브가 함께 재사용하도록 바꿨습니다
- Replay Studio를 단순 로그 뷰어가 아니라 결과 해석, 데이터 상태, 현재 포지션 의미를 보여주는 허브로 강화했습니다

## 왜 중요한가

예전 구조는 강력한 기능을 많이 품고 있었지만 여전히 “패널 벽”처럼 보였습니다. 이번 방향 전환은 제품을 시그널 워크스페이스로 다시 읽히게 만듭니다. 수집하고, 검토하고, 리플레이하고, 비교하고, 결정하는 흐름이 훨씬 분명해졌습니다.

동시에 데이터 낭비도 줄였습니다. 스냅샷, 연구, 백테스트를 위해 모은 데이터가 세션용 임시 상태로만 남지 않고 여러 허브가 공유하는 저장 자산으로 더 잘 재사용됩니다.

## 사용자 영향

- 제품 첫인상이 단순 모니터보다 분석/투자 워크플로우에 가까워졌습니다
- 데스크톱뿐 아니라 웹 경로에서도 리플레이와 스케줄러 액션을 실행할 수 있습니다
- 수집한 인텔리전스가 세션을 넘어 더 잘 남고, 브리핑/그래프/리플레이가 함께 활용할 수 있습니다
- 운영 상태를 active stage, lag, blocker, readiness로 더 쉽게 이해할 수 있습니다
- GitHub 공개 문서도 상속된 모니터 정체성보다 현재 제품 방향을 먼저 설명하게 됐습니다

## 아직 남아 있는 경계

- 외부 데이터 품질과 공급자 rate limit은 여전히 리플레이 품질을 흔들 수 있습니다
- 일부 깊은 구현 식별자는 상속된 코드 구조를 반영합니다
- GitHub 저장소의 짧은 설명 문구 자체는 저장소 파일만으로는 완전히 바꿀 수 없고, GitHub 인증이 필요합니다
