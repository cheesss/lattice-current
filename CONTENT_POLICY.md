# Content Policy

This project publishes code openly, but it does not treat all runtime content as freely redistributable.

## Third-Party News and Source Content

- Do not republish full third-party news articles or long excerpts in repository docs, screenshots, or the docs site
- Use short quotations only when legally appropriate and necessary
- Prefer links, titles, source attribution, and project-authored summaries

## Screenshot and Demo Policy

Public screenshots, GIFs, and docs images should be sanitized:

- remove or blur API keys, tokens, email addresses, machine names, and credentials
- avoid exposing internal-only sources, private feed names, or investigative infrastructure
- avoid long visible excerpts from copyrighted articles
- prefer product-authored charts, diagrams, and synthesized cards over raw third-party article text

## Data and Operational Details

The public repository and docs site intentionally omit or sanitize:

- private feed details
- proxy or bypass implementation specifics
- production-only operational procedures
- sensitive debugging output
- credentials and secret-handling details beyond high-level setup guidance

## AI Outputs

AI-generated summaries and analysis published in the docs site should:

- remain grounded in project behavior and architecture
- avoid fabricating source relationships or external results
- distinguish between project-authored analysis and third-party source material
